# Test Specs

This directory contains test plans generated by the Playwright Test Planner agent.

## Using Playwright Test Agents

### 1. Planner Agent
Creates comprehensive test plans by exploring your application:
```
Ask Claude Code to plan tests for a feature, e.g.:
"Create a test plan for the user authentication flow"
```
Output: Markdown test plan saved to `specs/`

### 2. Generator Agent
Converts test plans into executable Playwright tests:
```
Ask Claude Code to generate tests from a plan, e.g.:
"Generate tests from specs/auth-flow.md"
```
Output: Test files saved to `e2e/`

### 3. Healer Agent
Automatically repairs failing tests:
```
Ask Claude Code to heal failing tests, e.g.:
"Run tests and fix any failures"
```
Output: Updated test files with fixes

## Test Plan Format

Test plans should follow this structure:

```markdown
# Feature Name

**Seed:** `e2e/seed.spec.ts`

## 1. Test Group Name

### 1.1 Test Case Name
**Steps:**
1. Navigate to /path
2. Click the "Button" button
3. Fill in the "Email" field with "test@example.com"

**Expected:**
- User should see success message
- Page should redirect to /dashboard
```

## Project Conventions

- Use fixtures from `e2e/fixtures.ts` for authentication
- Tests run sequentially (workers: 1) due to SQLite database
- Use `authenticatedPage` fixture for tests requiring login
- Test file naming: `feature-name.e2e.ts`
